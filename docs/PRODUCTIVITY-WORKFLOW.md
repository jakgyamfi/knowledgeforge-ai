# Living Execution Plans

KnowledgeForge is designed to move ideas toward results, not merely store them. Every book, business, project, or general workspace can maintain a living execution plan beside its working document.

## Automatic planning

When new voice, text, document, or image material is successfully analyzed and integrated, the active AI provider reviews:

- the current working document;
- the workspace owner’s persistent direction;
- relevant source notes;
- existing task status; and
- the reason for the latest reorganization.

It then proposes a newly ordered set of open tasks with priority, dependencies, estimated effort, and a target date when a deadline is supported or a proposed schedule would be useful. The interface totals the expected time for all open work.

Dates and estimates are planning aids, not guarantees. The planner is instructed not to invent external commitments or unsupported deadlines.

## Protected task state

Automatic replanning is deliberately non-destructive:

- completed tasks remain completed;
- manually created tasks are retained;
- only open AI-generated suggestions are replaced;
- a completed task with the same title is not recreated; and
- failure to generate a plan never rolls back successfully integrated workspace content.

## Owner controls

From **Living execution plan** in the application, the owner can:

- add a task manually;
- supply an effort estimate and target date;
- mark work complete or reopen it; and
- request an immediate AI replan.

Future releases can add calendar synchronization, richer inline editing, dependency graphs, reminders, and historical plan revisions without changing the private source-ingestion model.
